# Dataset

## Primary dataset used by this notebook: real-world Civil Comments sample

`data/raw/train.csv` (**committed to this repo**, 80,000 rows, ~7.8% toxic)
is a real-world sample of the
[`google/civil_comments`](https://huggingface.co/datasets/google/civil_comments)
dataset on the Hugging Face Hub — the dataset behind the Jigsaw *"Unintended
Bias in Toxicity Classification"* competition. Each comment has a continuous
human-rated toxicity score in `[0, 1]`; we binarize `toxic = 1` when
`toxicity >= 0.5`.

It was generated by `data/prepare_real_dataset.py`. Regenerate (or resample
with different size/seed) any time with:

```bash
pip install datasets
python data/prepare_real_dataset.py
```

`text_toxicity_moderation.ipynb` automatically picks up `data/raw/train.csv`
if present and trains on it — this is what the current model/metrics in the
notebook are trained on.

### Why Civil Comments instead of the Kaggle Jigsaw Toxic Comment Classification CSV?

The original Jigsaw competition dataset is hosted on Kaggle and requires a
personal Kaggle account + API token — there is no anonymous/programmatic
download. Civil Comments is mirrored publicly on the Hugging Face Hub with
**no authentication required**, so `prepare_real_dataset.py` can reproduce
the exact training data for anyone who clones this repo with zero setup.

### Option — use the original Jigsaw CSV instead (if you have a Kaggle account)

- Source: https://www.kaggle.com/competitions/jigsaw-toxic-comment-classification-challenge/data
- ~160k Wikipedia talk-page comments, labelled `toxic`, `severe_toxic`,
  `obscene`, `threat`, `insult`, `identity_hate`.
- Steps:
  1. Create a free Kaggle account and accept the competition rules.
  2. `pip install kaggle`, place your `kaggle.json` API token in `~/.kaggle/`.
  3. `kaggle competitions download -c jigsaw-toxic-comment-classification-challenge -p data/raw`
  4. Unzip so the file lives at `data/raw/train.csv` (overwrites the Civil
     Comments version — that's fine, the notebook auto-detects the format).

The notebook derives `toxic` from the Jigsaw multi-label file as `1` if
**any** of `toxic/severe_toxic/obscene/threat/insult/identity_hate` is `1`,
else `0`.

## Fallback: offline sample dataset (100 rows)

`sample_toxicity_data.csv` (100 hand-written rows, 50 toxic / 50 non-toxic)
is generated by `generate_sample_data.py`. If `data/raw/train.csv` is ever
removed, the notebook falls back to this so it still runs top-to-bottom with
zero setup. **This tiny sample dataset is for pipeline demonstration only —
it is not large/diverse enough to train a reliable classifier** (an earlier
version of this project trained only on this sample and mis-flagged
"You are a wonderful person." as toxic — that's why the real 80,000-row
Civil Comments dataset above is now the default).

```bash
cd data
python generate_sample_data.py
```

## Expected column format

Whatever source you use, the notebook expects (or maps to) two columns:

| column         | meaning                              |
|----------------|---------------------------------------|
| `comment_text` | the raw text of the comment            |
| `toxic`        | binary label, `1` = toxic, `0` = clean |
