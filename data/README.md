# Dataset

## Included: offline sample dataset

`sample_toxicity_data.csv` (100 rows, 50 toxic / 50 non-toxic) is generated by
`generate_sample_data.py` and is committed to the repo so that
**`text_toxicity_moderation.ipynb` runs end-to-end with no internet access or
API keys required**. It's intentionally small and hand-curated (mild
insults/profanity, no slurs or hate speech) so it's safe to commit and share.

Regenerate it any time with:

```bash
cd data
python generate_sample_data.py
```

## Recommended: real-world dataset (for a production-quality model)

The sample dataset is only good for demonstrating the pipeline. For a model
you'd actually trust, download one of these public datasets and drop the CSV
into `data/raw/`:

### Option A — Jigsaw Toxic Comment Classification Challenge (recommended)
- Source: https://www.kaggle.com/competitions/jigsaw-toxic-comment-classification-challenge/data
- ~160k Wikipedia talk-page comments, labelled `toxic`, `severe_toxic`,
  `obscene`, `threat`, `insult`, `identity_hate`.
- Steps:
  1. Create a free Kaggle account and accept the competition rules.
  2. `pip install kaggle`, place your `kaggle.json` API token in `~/.kaggle/`.
  3. `kaggle competitions download -c jigsaw-toxic-comment-classification-challenge -p data/raw`
  4. Unzip `train.csv` into `data/raw/train.csv`.

### Option B — Jigsaw Unintended Bias in Toxicity Classification (Civil Comments)
- Source: https://www.kaggle.com/competitions/jigsaw-unintended-bias-in-toxicity-classification/data
- ~1.8M comments from the Civil Comments platform with a continuous
  `target` toxicity score (0-1) plus identity subgroup labels — useful for
  studying bias.

### Option C — Hugging Face Hub mirrors (no Kaggle account needed)
```python
from datasets import load_dataset
ds = load_dataset("google/civil_comments")          # Civil Comments
# or
ds = load_dataset("jigsaw_toxicity_pred")            # Jigsaw (may require manual download)
```

The notebook's data-loading cell automatically checks for
`data/raw/train.csv` first (real dataset) and falls back to
`data/sample_toxicity_data.csv` if it isn't found — so you can drop in the
real dataset later without changing any code.

## Expected column format

Whatever source you use, the notebook expects (or maps to) two columns:

| column         | meaning                              |
|----------------|---------------------------------------|
| `comment_text` | the raw text of the comment            |
| `toxic`        | binary label, `1` = toxic, `0` = clean |

If you load the Jigsaw multi-label file, the notebook derives `toxic` as
`1` if **any** of `toxic/severe_toxic/obscene/threat/insult/identity_hate`
is `1`, else `0`.
